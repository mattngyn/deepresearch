## Standard Evaluators

### Browser Evaluators

Common evaluation tools for web environments:

```python
# URL matching
@hub.register_evaluator("url_match")
def url_match(pattern: str, regex: bool = False) -> dict:
    """Check if current URL matches pattern"""
    
# Text presence
@hub.register_evaluator("page_contains")  
def page_contains(text: str, selector: str = None) -> dict:
    """Check if page contains text"""

# Element existence
@hub.register_evaluator("element_exists")
def element_exists(selector: str) -> dict:
    """Check if element exists on page"""

# Form validation
@hub.register_evaluator("form_filled")
def form_filled(fields: dict[str, str]) -> dict:
    """Check if form fields have expected values"""
```

### Data Evaluators

For spreadsheet/data environments:

```python
# Cell value check
@hub.register_evaluator("cell_equals")
def cell_equals(cell: str, expected: Any) -> dict:
    """Check if cell has expected value"""

# Formula validation  
@hub.register_evaluator("formula_correct")
def formula_correct(cell: str, expected_formula: str) -> dict:
    """Verify cell formula"""

# Data range check
@hub.register_evaluator("range_sum")
def range_sum(range: str, expected: float) -> dict:
    """Check sum of cell range"""
```

### Game Evaluators

For game environments:

```python
# Score threshold
@hub.register_evaluator("score_above")
def score_above(threshold: int) -> dict:
    """Check if score exceeds threshold"""

# Level completion
@hub.register_evaluator("level_complete")  
def level_complete(level: int) -> dict:
    """Check if level was completed"""

# Achievement check
@hub.register_evaluator("achievement_unlocked")
def achievement_unlocked(achievement_id: str) -> dict:
    """Check if achievement was earned"""
```

```python
@hub.register_setup("navigate_to_app")
def navigate_to_app(section: str = "home") -> dict:
    """Navigate to app section using playwright"""
    urls = {
        "home": "https://app.example.com", 
        "dashboard": "https://app.example.com/dashboard",
        "settings": "https://app.example.com/settings"
```

## Registry Pattern

HUD environments use registries to organize components:

```python
from hud_controller.setup import SetupRegistry
from hud_controller.evaluators import EvaluatorRegistry
from hud_controller.problems import ProblemRegistry

# Setup registry
setup_registry = SetupRegistry()
setup_registry.register("initialize", initialize_fn)
setup_registry.register("load_data", load_data_fn)

# Evaluator registry  
eval_registry = EvaluatorRegistry()
eval_registry.register("check_success", check_success_fn)
eval_registry.register("validate_output", validate_output_fn)

# Problem registry (task templates)
problem_registry = ProblemRegistry()
problem_registry.register("form_filling", FormFillingProblem)
problem_registry.register("navigation", NavigationProblem)
```

## Computer Tool Variants

Different environments may use different computer tool implementations:

### Anthropic Computer Tool

```json
{
  "name": "computer",
  "parameters": {
    "action": "click|type|screenshot|scroll|key",
    "coordinate": [x, y],
    "text": "text to type",
    "key": "key to press"
  }

```

### OpenAI Computer Tool

```json
{
  "name": "computer",
```

## Environment Capabilities

### Standard Capabilities

Most environments should provide:

```python
# Minimum viable environment
class MinimalEnvironment:
    tools = ["interact", "get_state"]
    setup_tools = ["reset"]
    evaluate_tools = ["check_complete"]
```

### Extended Capabilities

Advanced environments may add:

```python
# Full-featured environment
class AdvancedEnvironment:
    tools = [
        # Interaction
        "click", "type", "select", "drag",
        # Navigation  
        "playwright", "back", "forward", "refresh",
        # State
        "screenshot", "get_text", "get_html",
        # Advanced
        "execute_script", "upload_file", "download_file"
    ]
    
    setup_tools = [
        "reset", "load_scenario", "configure_options",
        "create_test_data", "import_fixtures"
    ]
    
    evaluate_tools = [
        "check_complete", "validate_data", "score_performance",
        "compare_output", "measure_efficiency"
    ]
    
    resources = [
        "logs://session", "state://current", "config://environment"
    ]
```

## Error Response Patterns

Standard error responses across environments:

```python
# Tool not found
{
    "error": {
        "code": "TOOL_NOT_FOUND",
        "message": "Tool 'unknown_tool' not found",
        "available_tools": ["click", "type", "screenshot"]
    }
}

# Invalid arguments
{
    "error": {
        "code": "INVALID_ARGS",
        "message": "Missing required parameter 'url'",
        "schema": {
            "type": "object",
            "properties": {"url": {"type": "string"}},
            "required": ["url"]
        }
    }
}

# Execution error
{
    "error": {
        "code": "EXECUTION_ERROR",
        "message": "Element not found: #submit-button",
        "recoverable": true,
        "suggestion": "Try using a different selector or wait for element"
    }
}
```

### Resource Limits

Recommended limits for stability:

- **Memory**: 4GB per environment instance
- **CPU**: 2 cores per instance
- **Disk**: 10GB temporary storage
- **Network**: 100 Mbps minimum
- **Concurrent tools**: 1 (serialize calls)
